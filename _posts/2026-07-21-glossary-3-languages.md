개발 용어 Q&A 세 번째 편. 대표 언어 셋을 비교해 봅니다.

## Q. Python은 어떤 언어예요?

**A.** 읽기 쉽고 배우기 쉬운 만능 언어예요. 영어 문장처럼 읽히고, 들여쓰기로 코드 블록을 구분해요. 특히 **AI·데이터 분석의 사실상 표준** 언어예요. 🐍

```python
age = 20
if age >= 18:
    print("성인입니다")
```

> 콜론(`:`)은 "여기부터 이 조건에 속하는 코드가 시작된다"는 신호예요.
{: .prompt-tip }

## Q. JavaScript는요?

**A.** 웹의 기본 프로그래밍 언어예요. 브라우저에서 **동작**을 담당하고(버튼 클릭 반응 등), Node.js로 브라우저 밖에서도 실행돼요. 웹페이지가 사람이라면 JS는 두뇌·근육이에요. 🧠

## Q. TypeScript는 JavaScript와 뭐가 달라요?

**A.** JavaScript에 **타입(type)**을 더한 언어예요. "이 변수는 숫자만 담아"라고 미리 약속해두면, 실수를 **작성 중에 미리** 잡아줘요.

```typescript
let age: number = 25;
age = "스물다섯"; // ❌ 작성 중에 바로 에러!
```

핵심은, TypeScript는 실행 전에 **JavaScript로 변환**된다는 거예요. 타입 정보는 변환 시 사라지고, 실제 실행되는 건 순수 JS예요. 그래서 결과물·성능은 사실상 같고, 차이는 대부분 **개발 경험**(안전함·자동완성)에 있어요.

| | JavaScript | TypeScript |
|---|-----------|-----------|
| 타입 | 없음 | 있음 (`: number`) |
| 오류 발견 | 실행 중 😰 | 작성 중 ✅ |
| 실행되는 것 | JS | JS (변환됨) |

> "TypeScript는 실행되는 언어가 아니라, 더 나은 JavaScript를 쓰기 위한 도구다."
