## Q. 객체(Object)가 뭐예요?

**A.** 관련된 **속성(정보)**과 **메서드(기능)**를 하나로 묶은 것이에요. `{ }` 안에 "키: 값" 형태로 정의하죠. 여러 칸으로 나뉜 상자라고 생각하면 돼요. 📦

```javascript
const dog = {
  name: "뽀삐",        // 속성
  bark: function() {   // 메서드 (값이 함수인 속성)
    console.log("멍멍!");
  }
};
```

> 속성만, 메서드만, 빈 객체 모두 가능해요. 메서드는 사실 "값이 함수인 속성"일 뿐이에요.
{: .prompt-tip }

## Q. const는 객체 전용 키워드인가요?

**A.** 아니요! **모든 변수 선언에 쓰는** 키워드예요. constant(상수)의 약자로, 한 번 정한 값을 **재할당할 수 없다**는 뜻이에요.

주의할 점: `const`로 만든 객체는 **재할당은 안 되지만 내부 속성은 바꿀 수 있어요.**

```javascript
const dog = { name: "뽀삐" };
dog.name = "초코";        // ✅ 가능 (속성 변경)
dog = { name: "바둑이" };  // ❌ 에러 (객체 통째 교체)
```

> const는 "주소가 고정된 집"이에요. 집 내부 인테리어는 바꿔도, 이사(재할당)는 못 해요.
{: .prompt-info }

## Q. 추상화(Abstraction)는요?

**A.** 복잡한 내부는 숨기고 **필요한 핵심만 보여주는 것**이에요. 자동차를 운전할 때 핸들·페달만 조작하면 되고 엔진 내부는 몰라도 되는 것처럼요. 🚗

사실 우리가 쓰는 함수, API, 라이브러리가 다 추상화예요. 컴퓨터는 추상화가 겹겹이 쌓인 구조예요: **내 코드 → 언어 → 커널 → 기계어**. 위 층에서 쉬운 명령만 내리면 아래 복잡한 층이 알아서 처리하죠.

> 추상화는 프로그래밍적 사고(Computational Thinking)의 4대 요소 중 하나예요. (분해·패턴인식·추상화·알고리즘)
