## Q. 터미널(Terminal)이 뭐예요?

**A.** 글자를 보여주고 키 입력을 전달하는 **창**이에요. 원래 terminal은 통신에서 "선로의 끝단(말단 장치)"을 뜻했어요. 옛날엔 대형 컴퓨터 한 대를 여러 명이 나눠 썼는데, 그때 키보드+화면으로 접속하던 물리 장치가 터미널이었죠. 지금의 macOS Terminal, VS Code 터미널은 그걸 화면 속에서 흉내 낸 **에뮬레이터**예요.

> 터미널은 글자를 주고받을 뿐, 명령의 뜻은 **해석하지 않아요.** (식당의 주문 창구)
{: .prompt-tip }

## Q. 셸(Shell)은요?

**A.** 사람이 친 명령을 **해석해서** 프로그램을 실행하는 프로그램이에요. 운영체제 핵심(커널)을 감싼 "껍데기"라서 셸(shell)이라 불러요. bash, zsh 같은 여러 종류가 있어요.

> 비유하면 **웨이터**예요. "김치찌개 주세요"를 알아듣고 주방에 전달하죠.
{: .prompt-info }

## Q. 커널(Kernel)은 뭐가 다른가요?

**A.** 운영체제의 **핵심**이에요. 프로그램이 CPU·메모리·디스크 같은 자원을 쓰도록 **중개·통제**하죠. 셸이 껍데기라면 커널은 알맹이예요. 직접 요리하기보다, 요리사(프로그램)들에게 화구·재료를 누가 쓸지 배분하는 **주방장**에 가까워요. 👨‍🍳

## Q. 프롬프트($, %)는 명령어인가요?

**A.** 아니요! 셸이 **"입력을 기다린다"**고 보여주는 표시예요. `%`는 zsh, `$`는 bash(일반 사용자), `#`은 관리자(root)를 뜻해요. 문서에 `% pwd`라고 적혀 있으면 `%`는 예시 표시일 뿐, 실제로 칠 건 `pwd`뿐이에요.

> `#`이 보이면 관리자 권한 상태예요. 신중히!
{: .prompt-warning }

## Q. bin과 PATH는요?

**A.** `bin`은 실행 파일(binary)이 담긴 폴더, `PATH`는 명령어를 찾을 때 뒤질 bin 폴더들의 **목록**이에요. bin이 연장통이라면 PATH는 "어느 연장통을 열어볼지 정한 목록"이에요. 🧰

## Q. 자주 쓰는 명령어 몇 개만 정리하면?

**A.**

| 명령어 | 하는 일 |
|--------|---------|
| `pwd` | 현재 폴더 경로 출력 ("나 어디 있지?") |
| `echo` | 입력·변수 값을 그대로 출력 (`echo $SHELL`) |
| `sudo` | 관리자 권한으로 명령 실행 (마스터 키 🔑) |

> `sudo`는 시스템 거의 전부를 바꿀 수 있어요. 뜻 모르는 명령에 함부로 붙이지 마세요.
{: .prompt-danger }

## Q. zsh와 bash 차이는? LTS는 무슨 뜻이에요?

**A.** 둘 다 셸이에요. bash는 오래된 표준, **zsh**는 요즘 Mac 기본(자동완성·꾸미기가 강력)이에요. 그리고 **LTS**(Long-Term Support)는 "오래·안정적으로 지원되는 버전"을 뜻해요 — 검증되고 정비 지원이 오래 보장되는 자동차 모델 같은 거죠. 🚗
