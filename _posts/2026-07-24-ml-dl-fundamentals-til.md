오늘은 LLM이 "어떻게 이해하고 학습하는지" 그 속사정을 파고들었어요. 데이터를 숫자로 바꾸는 단계부터, 신경망이 스스로 똑똑해지는 원리, 그리고 그걸 실전에 응용하는 방법까지 순서대로 정리했어요.

## Q. 인공지능·머신러닝·딥러닝·생성형 AI는 뭐가 다른가요?

**A.** 이 넷은 경쟁 관계가 아니라 **포함 관계**예요. `인공지능 ⊃ 머신러닝 ⊃ 딥러닝 ⊃ 생성형 AI` — 마트료시카처럼 큰 개념 안에 작은 개념이 들어있어요.

- **인공지능(AI)**: 기계를 똑똑하게 행동하게 만드는 모든 기술 (아주 넓음)
- **머신러닝**: 규칙을 사람이 안 짜고, 데이터로 기계가 스스로 학습
- **딥러닝**: 인공 신경망을 여러 겹(층) 깊게 쌓아 학습
- **생성형 AI**: 딥러닝으로 "새로운 것을 창조" — 여기에 LLM(Claude, GPT)이 있어요

> 요리사에 비유하면: AI=요리 잘하는 사람 → 머신러닝=맛보며 배우는 요리사 → 딥러닝=감각을 깊게 훈련한 요리사 → 생성형 AI=배운 걸로 새 요리를 창작하는 셰프.
{: .prompt-tip }

## Q. 토큰·벡터·임베딩이 뭐예요?

**A.** LLM이 글을 "숫자로 이해"하는 3단계예요. 컴퓨터는 글자를 원래 못 읽고 숫자만 계산할 수 있어서, 이 변환이 꼭 필요해요.

1. **토큰**: 글을 잘게 자른 조각 (레고 블록 하나하나)
2. **임베딩**: 토큰을 벡터로 바꾸는 변환 과정
3. **벡터**: 그 결과인 숫자들의 목록 (의미를 담은 좌표)

```
"고양이" → 임베딩 → [0.8, 0.1, -0.3, ...]
"강아지" → 임베딩 → [0.7, 0.15, -0.25, ...]  # 고양이와 벡터가 비슷!
```

> 핵심 마법: **의미가 비슷한 단어는 벡터 공간에서 가까운 위치**에 놓여요. AI가 "고양이와 강아지가 비슷한 개념"이라는 걸 "이해"하는 방식의 뿌리예요.
{: .prompt-info }

API 요금이 "토큰당 과금"인 이유도 이 토큰 개수를 기준으로 매기기 때문이에요.

## Q. 스칼라·벡터·행렬·텐서는 또 뭔가요?

**A.** 이건 "숫자를 몇 겹으로 쌓았는가"의 사다리예요.

| 이름 | 차원 | 생김새 | 예시 |
|------|------|--------|------|
| 스칼라 | 0차 | 점 | `5` |
| 벡터 | 1차 | 한 줄 | `[3, 5, 7]` |
| 행렬 | 2차 | 표 | `[[1,2],[3,4]]` |
| 텐서 | 3차+ | 표를 여러 겹 | 행렬을 여러 장 쌓음 |

> PyTorch·TensorFlow 이름에 "텐서"가 들어있는 이유가 바로 이거예요 — 딥러닝 계산은 대부분 텐서 단위로 이뤄지거든요.
{: .prompt-tip }

## Q. 퍼셉트론, bias는 뭔가요?

**A.** **퍼셉트론**은 신경망의 최소 단위(뉴런 하나)예요. 여러 입력에 각각 중요도(가중치)를 곱해 더하고, **bias**(편향)라는 "기본 성향 값"을 더한 뒤, 기준을 넘으면 다음으로 신호를 보내요.

```
결과 = (입력1 × 가중치1) + (입력2 × 가중치2) + ... + bias
```

> 미니 투표소처럼 생각하면 쉬워요. "나들이 갈까?"를 날씨(+3표)·시간있음(+2표)·피곤함(-1표)으로 합산해서 기준을 넘으면 "간다!"고 판단해요. bias는 이 투표의 "기본 성향"(낙관적/신중함)을 조절하는 값이에요.
{: .prompt-tip }

## Q. 미분·체인룰·역전파는 어떻게 연결되나요?

**A.** 신경망이 "학습"하는 절차의 3단계예요.

- **미분**: 지금 값을 살짝 바꾸면 결과가 어느 방향으로 변하는지 계산 (안개 속에서 발밑 경사를 느끼는 것)
- **체인룰**: 여러 층을 거친 영향력은 각 단계 영향력을 곱하면 된다는 규칙 (도미노 연쇄)
- **역전파**: 체인룰로 오차를 출력에서 입력 쪽으로 거꾸로 전파하며, 모든 층의 가중치·bias를 한 번에 조정하는 알고리즘

> 프로젝트 실패 원인을 거꾸로 되짚는 회의라고 생각하면 돼요. 최종 발표팀 → 기획팀 → 조사팀 순으로 책임을 거슬러 올라가며 "다음엔 이렇게 고쳐"라고 피드백을 주는 것과 같아요.
{: .prompt-info }

이 반복을 수천~수만 번 하면서 신경망이 점점 똑똑해져요.

## Q. torch(PyTorch)와 autograd는 왜 필요해요?

**A.** 신경망 층이 수백 개면, 사람이 손으로 미분·체인룰을 계산하는 건 사실상 불가능해요. **PyTorch**는 텐서 계산과 신경망 구성을 쉽게 해주는 파이썬 도구고, **autograd**는 그 안에서 미분·역전파를 **자동으로** 계산해주는 기능이에요.

```python
x = torch.tensor(3.0, requires_grad=True)
y = x ** 2
y.backward()      # 자동 미분(역전파) 실행
print(x.grad)      # 6.0  (y=x²의 미분은 2x, x=3이니 6!)
```

> autograd는 계산 과정을 몰래 "녹화"해뒀다가, `.backward()` 한 줄로 거꾸로 되감으며 자동 미분해요. 자동 되감기 캠코더 같은 거예요. 📹
{: .prompt-tip }

## Q. 회귀 분석, softmax, MNIST는 어떤 관계예요?

**A.** 셋 다 "예측·분류"를 다루지만 성격이 달라요.

- **회귀 분석**: 데이터로 직선(관계식)을 찾아 **숫자**를 예측 (`y = 가중치×x + bias` — 사실 퍼셉트론 계산식과 같아요!)
- **softmax**: 여러 후보 중 뭐가 맞는지, 원시 점수를 **총합 100%인 확률**로 바꿔주는 함수 (분류에 씀)
- **MNIST**: 0~9 손글씨 숫자 이미지 7만 장 데이터셋 — 딥러닝의 "Hello World"

> 회귀는 "집값이 얼마?"처럼 숫자를 답하고, softmax는 "이 손글씨가 7일 확률 91%"처럼 여러 후보 중 확신도를 %로 보여줘요.
{: .prompt-info }

## Q. 인터프리터는 뭐예요?

**A.** 코드를 **한 줄씩 읽으며 바로 실행**하는 방식(또는 그 프로그램)이에요. 반대로 컴파일러는 전체를 미리 번역해두고 실행해요.

| | 인터프리터 | 컴파일러 |
|---|-----------|---------|
| 방식 | 한 줄씩 즉시 실행 | 전체를 미리 번역 후 실행 |
| 예시 | Python | TypeScript(→JS로 컴파일) |
| 오류 발견 | 실행 중 | 실행 전(번역 시점) |

> 동시통역사(인터프리터) vs 책 전체를 미리 번역하는 번역가(컴파일러)라고 생각하면 딱 맞아요.
{: .prompt-tip }

## Q. 파인튜닝과 전이학습은 뭐가 다른가요?

**A.** **전이학습**은 "한 작업에서 배운 지식을 비슷한 다른 작업에 재활용하자"는 큰 개념이고, **파인튜닝**은 그걸 실현하는 구체적 방법 — 이미 학습된 모델을 내 작업에 맞게 미세 조정하는 거예요.

> 기성복(이미 학습된 모델)을 사서 내 몸에 맞게 살짝 수선(파인튜닝)하는 것과 같아요. 처음부터 새 옷을 짓는 것보다 훨씬 빠르고 저렴해요. 참고로 RLHF도 파인튜닝의 한 종류예요.
{: .prompt-tip }

## Q. "이미 학습했다"는 게 저와 나누는 대화도 포함되나요?

**A.** 아니요! 이건 자주 헷갈리는 부분이에요. **학습(training)**은 서비스가 나오기 **전에**, 별도로 미리 끝난 과정이에요. 지금 저와 나누는 대화는 **추론(inference)** 단계 — 이미 고정된 가중치로 계산만 할 뿐, 대화한다고 실시간으로 똑똑해지지 않아요.

> 시험이 끝난 뒤 답안을 쓰는 것과 같아요. 공부(학습)는 이미 끝났고, 시험 시간(추론)엔 저장된 지식을 꺼내 쓸 뿐이에요. 대화 중 이전 내용을 "기억"하는 것처럼 보이는 건, 학습이 아니라 프롬프트로 이전 대화를 매번 통째로 다시 넣어주기 때문이에요.
{: .prompt-warning }

## Q. RAG는 이 모든 게 실전에서 어떻게 쓰이는 예인가요?

**A.** 맞아요! **RAG(검색 증강 생성)**는 AI가 답하기 전에 먼저 관련 자료를 검색해서, 그걸 참고해 답하게 하는 기술이에요. 앞서 배운 임베딩·벡터가 "의미로 검색하기"에, 그리고 LLM이 "찾은 자료로 답 생성하기"에 쓰이는 완성형 응용 사례예요.

1. 질문을 벡터로 변환(임베딩)
2. 벡터 데이터베이스에서 의미가 비슷한 문서 검색
3. 찾은 문서를 LLM에게 참고자료로 함께 제공
4. LLM이 그 내용을 보고 답 생성

> 오픈북 시험이라고 생각하면 쉬워요. 외운 것만으로 답하는 대신, 책(자료)을 펼쳐보고 정확하게 답하는 거예요. 할루시네이션(AI가 그럴듯하게 지어내는 것)을 줄이는 핵심 해결책이에요.
{: .prompt-tip }
